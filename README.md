# 프로젝트 관리·통합 대시보드 시스템

매년 운영하는 10~20개의 인재양성/지원 사업(프로젝트)을
**선발 → 지급 → 성장관리 프로그램 → 성장관리 → 만족도** 프로세스로 관리하고,
분산된 데이터를 하나의 DB로 통합하여 전체·프로젝트별 KPI를 대시보드로 보여주는 웹 시스템입니다.

## 주요 기능

- **전체 대시보드**: 프로젝트 **상위 유형별 구분**(자유 입력·자동 그룹화), 유형별/유형 통합 **집행률**(총예산·지원금)·**집행잔액**, **월별 지원금 지급·집행 추이**(지급/반환 + 누적 집행률), 교육적 배려대상·학교급 통합 집계, 연도·유형 필터
- **프로젝트 상세**: 5단계 프로세스 파이프라인 + KPI 타일 + 탭(선발 세부구성·**지원금 집행 요약(최초/추가/반환/실집행/잔액)**·용도별/재원별 예산·프로그램·성과·**항목별 만족도**·협력기관)
- **데이터 관리**: 프로젝트(예산 세분·재원·유형), **지원금 내역(최초지급·추가지급·반환·사유·일자·최초선발인원)**, 협력기관, 만족도 항목별 점수를 담당자가 직접 입력 + **엑셀·CSV 통합 업로드**
- **감사 로그(레코드별)**: 모든 레코드에 **작성자·작성일시·수정자·수정일시** 자동 기록(관리 화면에 표시). 이름 스냅샷으로 저장되어 **담당자 변동·계정 삭제와 무관하게 데이터·기록이 그대로 보존**되며, 프로젝트의 `담당자` 필드와 독립적입니다.
- **감사 이력(변경 로그)**: 생성·수정·삭제·업로드를 시간순으로 누적하는 별도 이력([데이터 관리 → **감사 이력**] 탭). 누가/언제/무엇을/**어떻게(필드별 변경 전→후)** 바꿨는지 기록하며, 프로젝트·종류·작업으로 필터. 레코드가 삭제되어도 이력은 남습니다(append-only, `audit_logs` 테이블).
- **로그인 / 역할**: 관리자·담당자 구분, JWT 인증

**지원금 집행률 산식**: 실집행액 = 최초지급 + 추가지급 − 반환 / 집행률 = 실집행액 ÷ 지원금 총예산 × 100 /
집행잔액 = 지원금 총예산 − 실집행액(= 총예산 − 지급 − 추가지급 + 반환).

## 관리 KPI

- **선발**: 선발인원 · 경쟁률 · 세부구성(성별·학교급·지역) · **교육적 배려대상**(2종 이상, 통합 관리)
  - 학교급: 중학생·고등학생·대학생·대학원생(석사/박사)·박사후과정·대안학교(중등/고교)·학교밖청소년
- **예산·집행**: 용도별(총예산·지원금·심사관리비·프로그램운영비) / 재원별(출연금·운용소득·전기이월금·기부금·대행사업비)
  - 집행률: 유형별 총예산 대비 · 유형별 지원금 대비 · 유형 통합 총예산/지원금 대비
- **성장관리**: 프로그램 참여율 · 프로그램 만족도 · 성과 달성률
- **만족도**: 전체 만족도(**항목별 점수**) · 협력기관 현황

## 기술 스택

- **백엔드**: Python · FastAPI · SQLAlchemy · pandas/openpyxl(엑셀) · JWT
- **프론트엔드**: React · TypeScript · Vite · Recharts
- **DB**: 기본 SQLite(설치 불필요), 운영 시 PostgreSQL로 전환 가능

## 실행 방법 (Windows)

> Python 3.12+ 와 Node.js LTS 가 설치되어 있어야 합니다.
> (없다면: `winget install Python.Python.3.12` , `winget install OpenJS.NodeJS.LTS`)

### 1) 백엔드

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env          # 최초 1회
python -m app.seed              # (선택) 데모 데이터 5개 프로젝트 생성
uvicorn app.main:app --reload   # http://localhost:8000  (API 문서: /docs)
```

### 2) 프론트엔드 (새 터미널)

```powershell
cd frontend
npm install
npm run dev                     # http://localhost:5173
```

브라우저에서 **http://localhost:5173** 접속 → 로그인
(기본 관리자: 아이디 `admin`, 비밀번호 `admin1234` — 운영 전 반드시 변경).

## 엑셀/CSV 업로드 형식

[데이터 관리 → 엑셀·CSV 가져오기]에서 대상 프로젝트와 데이터 종류를 고른 뒤 업로드합니다.
헤더(첫 행) 이름으로 컬럼을 자동 매칭하며, 아래 헤더를 인식합니다(한글/영문 모두 가능).

| 종류 | 인식 헤더 예시 |
|---|---|
| 선발자 명단 | 이름, 성별, 연령대, 학교급, 지역, 소속, 배려대상, 선발일, 상태 |
| 지급 내역 | 예산항목(지원금/심사관리비/프로그램운영비), 지급유형, 계획금액, 지급액, 지급일, 상태 |
| 협력기관 | 기관명, 유형, 협력내용, 담당자, 협약시작, 협약종료, 상태 |
| 성장관리 성과 | 지표명, 목표값, 실적값, 측정일 |
| 만족도 설문 | 설문유형, 제목, 응답수, 평균점수, 실시일 |

> **배려대상**은 한 칸에 쉼표로 여러 개 입력 가능 (예: `기초생활수급, 한부모가정`).
> **예산항목**은 `지원금`/`심사관리비`/`프로그램운영비` 중 하나로 입력하면 용도별 집행률에 반영됩니다.

## Vercel 배포

프론트엔드(정적) + FastAPI(서버리스) + Neon PostgreSQL 구성으로 Vercel에 배포할 수 있습니다.
설정 파일(`vercel.json`, `api/index.py`, 루트 `requirements.txt`)이 준비되어 있으며,
단계별 방법은 **[DEPLOY.md](DEPLOY.md)** 를 참고하세요.

## 운영 배포 (PostgreSQL)

`backend/.env` 의 `DATABASE_URL` 을 PostgreSQL 연결 문자열로 변경하면 됩니다.
`docker-compose.yml` 로 PostgreSQL + 백엔드 + 프론트엔드를 함께 구동할 수 있습니다.
`SECRET_KEY` 와 최초 관리자 비밀번호는 운영 전 반드시 교체하세요.
